## Question 1 — Divide-and-Conquer

The Divide-and-Conquer algorithm breaks the problem into several sub-problems that are similar to the original problem but smaller in size, solves the sub-problems recursively, and then combines these solutions to create a solution to the original problem.

**(a)** *(3 marks)*  
Given a list of numbers of size *n*. Using a Divide-and-Conquer algorithm, you are required to find the product of all the numbers. Write a brief pseudocode.  
For example, for the following list [1, 2, 3, 8, 9], the product of all numbers *n* are: 1 × 2 × 3 × 8 × 9 = 432

**(b)** *(3 marks)*  
Given the list [2, 6, 7, 5], show how your algorithm in question 1(a) works, including the changes in the variables for each recursion. State the start, end and mid indexes for each step, and the return value for each recursive call.

**(c)** *(2 marks)*  
The time complexity function for the pseudocode in question 1(a) can be written as:  
T(n) = aT(n/b) + O(n^k · log_p n).  
Find the values for *a*, *b*, *k* and *p*.

**(b)** *(1 mark)*  
Define collision in a hash table.

**(c)** *(2 marks)*  
Describe a load factor (α) and its expected length, with the assumption of simple uniform hashing.

**(d)** *(6 marks)*

> H₁(k) = k % m  
> H₂(k) = 7 − (k % 7)

## Question 3 — Order Statistics

The *i*-th order statistic of a set of *n* elements is the *i*-th smallest element.

**(a)** *(2 marks)*  
How many comparisons are necessary to determine the minimum of a set of *n* elements?

**(b)** *(4 marks)*  
Given a set of *n* numbers, we wish to find the *i* largest in sorted order using a comparison-based algorithm. State the algorithm that sorts the numbers and lists the *i* largest with the best asymptotic worst-case running time. Based on your answer, what is its worst-case running time?

**(c)** *(2 marks)*  
Given a set of *n* numbers:  
**-6, 1, 4, 7, 12, 14, 18, 19, 28**  
Using order statistic, what is the best asymptotic worst-case running time to find the number at 7th position?
